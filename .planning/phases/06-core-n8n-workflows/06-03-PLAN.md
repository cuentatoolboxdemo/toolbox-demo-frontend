# Phase 6 Plan 3: n8n Chat Workflow

## Objective
Provide the RAG execution engine JSON and wire the Next.js frontend to securely pass the System Prompt, User query, and Memory.

## Steps

1. **Create `n8n/workflows/03-Chat.json`**:
   - Generate n8n Advanced AI nodes:
     - `Webhook` node: POST `/webhook/chat`. Expects `question`, `history` array, `system_prompt`.
     - `Conversational Retrieval QA Chain`: Master node joining all pieces.
     - `Window Buffer Memory`: To parse the incoming `history`.
     - `Qdrant Vector Store`: To search semantics. Attached to `Embeddings OpenAI`.
     - `OpenAI Model` node (configured for OpenRouter): The core LLM. Injects the `system_prompt`.

2. **Update `src/app/api/chat/route.ts`**:
   - Ensure `fetch()` sends `{ question, history, system_prompt }` to the `CHAT_WEBHOOK_URL`.
   - Parse the nested n8n response (`data[0].output` or similar) to extract the plain markdown string and return it as `{ answer: string }` to the Next.js frontend.

## Verification
- User imports JSON, maps OpenRouter and Qdrant.
- Chat from Next.js sends everything.
- AI answers natively referencing the documents found in Qdrant.
