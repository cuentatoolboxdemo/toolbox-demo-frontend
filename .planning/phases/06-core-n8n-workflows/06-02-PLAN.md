# Phase 6 Plan 2: n8n Delete Workflow

## Objective
Provide the JSON workflow for the context cleanup and wire the Next.js `DELETE` route to effectively trigger it.

## Steps

1. **Create `n8n/workflows/02-Delete.json`**:
   - Write a complete n8n workflow JSON structure containing:
     - `Webhook` node: POST `/webhook/delete`, receives `document_id`.
     - `S3` node: Delete file from `toolbox-demo:documents`. (Optional/If provided filename).
     - `HTTP Request` node (Qdrant API): The native Qdrant node in n8n only does Insert/Search. We need a `DELETE /collections/our_collection/points/delete` payload using Qdrant's Filter API: `{"filter": {"must": [{"key": "document_id", "match": {"value": "{{$json.body.document_id}}"}}]}}`.

2. **Update `src/app/api/docs/[id]/route.ts`**:
   - Ensure the Next.js backend passes the `document_id` JSON payload to the `DELETE_WEBHOOK_URL` securely.

## Verification
- User imports JSON to n8n, maps Qdrant URL/API Key on the HTTP node.
- User clicks Delete on the Admin UI.
- Webhook fires, Qdrant Vectors are expunged based on the matching ID.
