# Phase 6 Plan 1: n8n Ingest Workflow

## Objective
Provide the copy-paste JSON logic for n8n to ingest PDFs into Qdrant via MinIO, and wire the Next.js backend to push the actual files as form-data rather than just mocking it.

## Steps

1. **Create `n8n/workflows/01-Ingest.json`**:
   - Write a complete n8n workflow JSON structure containing:
     - `Webhook` node: POST, Auth none, listens on `/webhook/ingest`.
     - `S3` node: Upload file to `toolbox-demo:documents`.
     - `Read PDF` node: Parses the binary data from the webhook into text.
     - `Text Splitter` node: Splits text by sentences, max chunk size 1000.
     - `Embeddings OpenAI` node: Configured with OpenRouter credentials.
     - `Qdrant` node: Upload Document (insert vectors), injects metadata payload `document_id` mapped from the Webhook incoming body.

2. **Update `src/app/api/ingest/route.ts`**:
   - Locate the `try`/`catch` block around the mock processing.
   - Force it to *actually* send the file binary to the `INGEST_WEBHOOK_URL` containing `{ file: File, document_id: string }`.
   - Keep the local `docs.json` append for the UI visual collection list to work smoothly.

## Verification
- User imports JSON to n8n.
- User sets MinIO, Qdrant, and OpenRouter credentials on the imported nodes.
- User uploads a PDF in the Next.js UI; it triggers the webhook, Qdrant stores the vectors.
