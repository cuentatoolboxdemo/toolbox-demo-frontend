<task type="auto">
  <name>Task 2: Install and configure Shadcn UI, Lucide React, and next-pwa</name>
  <files>components.json, src/lib/utils.ts, next.config.mjs, package.json</files>
  <action>
    1. Install additional dependencies:
       `npm install lucide-react next-pwa`
       `npm install -D @types/node`
       `npx shadcn@latest init` — when prompted, select: New York style, Zinc base color, CSS variables yes. This creates `components.json` and `src/lib/utils.ts` with the `cn()` helper.

    2. Wrap next.config.mjs with next-pwa. Replace the file content with:
    ```js
    import withPWAInit from "next-pwa";

    const withPWA = withPWAInit({
      dest: "public",
      disable: process.env.NODE_ENV === "development",
      register: true,
      skipWaiting: true,
    });

    const nextConfig = {
      reactStrictMode: true,
    };

    export default withPWA(nextConfig);
    ```

    NOTE: next-pwa in development mode is disabled (`disable: process.env.NODE_ENV === "development"`) — this is intentional. The service worker only registers in production builds. The install prompt still fires in dev if manifest.json is valid.

    3. Verify Shadcn initialized correctly: `components.json` should exist with `"style": "new-york"` and `"baseColor": "zinc"`. `src/lib/utils.ts` should export a `cn()` function using `clsx` and `tailwind-merge`.
  </action>
  <verify>

    3. `cat next.config.mjs` — contains `withPWA`.
    4. `npm run build` succeeds (no TypeScript or compilation errors). Accept next-pwa service worker warnings as non-fatal.
  </verify>
  <done>
