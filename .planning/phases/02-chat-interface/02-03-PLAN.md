<task type="auto">
  <name>Task 1: Implement webhook POST with loading state and response rendering</name>
  <files>
    src/components/chat/ChatInterface.tsx
  </files>
  <action>
    Replace the `// TODO(02-03)` stub in `ChatInterface.tsx` with real fetch logic.

    **Webhook payload schema (CHAT-07):**
    ```ts
    type WebhookPayload = {
      question: string;      // the user's current message text
      tenant: string;        // tenant.slug (e.g. "calzedonia")
      history: Array<{       // OpenAI-style conversation history (all prior messages, NOT the current question)
        role: "user" | "assistant";
        content: string;
      }>;
      systemPrompt: string;  // always present — read from localStorage key "systemPrompt", fallback to ""
    };
    ```

    **`handleSend` implementation:**
    1. Build `userMessage` and append to `messages` state immediately (already done in Plan 02)
    2. Set `isLoading(true)`
    3. Build `history` from current `messages` (the array BEFORE appending userMessage) — map each to `{ role, content }`
    4. Read `systemPrompt` from `localStorage.getItem("systemPrompt") ?? ""`
    5. `fetch(process.env.NEXT_PUBLIC_CHAT_WEBHOOK_URL!, { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ question: text, tenant: tenant.slug, history, systemPrompt }) })`
    6. Parse response as JSON — expect `{ answer: string }` from the webhook. If the key is missing, fall back to `response.text()`.
    7. Build `assistantMessage: Message` with `role: "assistant"` and `content: answer`
    8. Append assistant message to state: `setMessages(prev => [...prev, assistantMessage])`
    9. Set `isLoading(false)` in a `finally` block
    10. On fetch error: append an assistant message with content "Sorry, something went wrong. Please try again." — do NOT crash the component

    **Loading indicator in MessageList:**
    Pass `isLoading: boolean` as a prop to `MessageList`. When `isLoading` is true, render a pulsing loading bubble after the last message:
    ```tsx
    <div className="mr-auto max-w-[80%] bg-muted text-muted-foreground rounded-2xl rounded-bl-sm px-4 py-2 text-sm animate-pulse">
      Thinking...
    </div>
    ```
    Update `MessageList.tsx` to accept `{ messages: Message[]; isLoading?: boolean }` and render the loading bubble conditionally.

    **localStorage safety:** `localStorage` is not available during SSR. Wrap all `localStorage` reads in a `typeof window !== "undefined"` check, or use a `useEffect`-initialized ref. Since ChatInterface is "use client", `typeof window !== "undefined"` check in the function body is safe.
