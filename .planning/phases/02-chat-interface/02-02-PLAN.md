  <action>
    Both are "use client" components. Define the shared `Message` type inside `MessageList.tsx` and re-export it from there (ChatInterface will import it).

    **`src/components/chat/MessageList.tsx`**

    Define and export:
    ```ts
    export type Message = {
      id: string;        // crypto.randomUUID() or Date.now().toString()
      role: "user" | "assistant";
      content: string;
      timestamp: Date;
    };
    ```

    Render a scrollable list of messages:
    - Container: `flex flex-col gap-2 overflow-y-auto flex-1 p-4`
    - User messages: `ml-auto max-w-[80%] bg-primary text-primary-foreground rounded-2xl rounded-br-sm px-4 py-2 text-sm`
    - Assistant messages: `mr-auto max-w-[80%] bg-muted text-foreground rounded-2xl rounded-bl-sm px-4 py-2 text-sm`
    - Empty state: render a centered muted paragraph "Send a message to get started"
    - Auto-scroll to bottom on new messages: use a `useEffect` with a `ref` on the container div and call `scrollIntoView({ behavior: "smooth" })` on the last message element

    **`src/components/chat/MessageInput.tsx`**

    Props: `{ onSubmit: (text: string) => void; disabled?: boolean }`

    - Controlled input using `useState<string>("")`
    - Render a `<form>` with `onSubmit` handler that calls `props.onSubmit(value.trim())` then clears the input
    - Prevent submitting empty strings
    - Layout: `flex gap-2 p-4 border-t border-border bg-background`
    - Input: Shadcn `Input` component (add it via `npx shadcn@latest add input` — this adds `src/components/ui/input.tsx`)
    - Button: Shadcn `Button` with `Send` text and `type="submit"`, disabled when `disabled` prop is true or input is empty
    - Also submit on Enter key (default form submit handles this — no extra keydown handler needed)
  </action>

  </verify>
