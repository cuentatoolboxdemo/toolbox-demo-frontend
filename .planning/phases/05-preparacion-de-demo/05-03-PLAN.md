# Phase 5 Plan 3: Mobile Viewport Optimization

## Objective
Ensure the chat input field is always comfortably above the mobile keyboard and never cut off by the browser address bar.

## Steps

1. **Refactor ChatInterface Layout**:
   - Open `src/components/chat/ChatInterface.tsx`.
   - Update the outer `div` className from `h-[calc(100vh-64px)]` to `flex flex-col flex-1 h-full overflow-hidden`.
   - Ensure the `MessageList` has `flex-1 overflow-y-auto`.
   - Ensure `MessageInput` is visually bound to the bottom without being pushed out of the flex container.

2. **Refactor Root Page Layout**:
   - Open `src/app/page.tsx`.
   - Ensure the outer wrapper uses `h-[100dvh]` instead of `min-h-screen` or `100vh`. 
   - A standard pattern is `<main className="flex flex-col h-[100dvh]">` where the header is fixed or flex, and the `<ChatInterface>` fills the remaining height `flex-1 overflow-hidden`.

## Verification
- Run `npm run dev`. On desktop Chrome, open Device Toolbar (Mobile View).
- Verify there are no double scrollbars.
- Verify the input area stays fixed at the bottom, and scrolling the chat history does not push the input area out of view.
- (Manual true validation requires opening the dev tunnel on a physical iOS/Android phone).
