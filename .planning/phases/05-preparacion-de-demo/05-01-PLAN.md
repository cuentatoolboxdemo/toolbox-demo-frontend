# Phase 5 Plan 1: Document Deletion (Context Isolation)

## Objective
Allow the user to delete documents from the active RAG context to prep the application for a fresh client demo.

## Steps

1. **Create DELETE API Route**:
   - Create `src/app/api/docs/[id]/route.ts`.
   - Add a `DELETE` export.
   - Extract the `id` from the URL params.
   - Verify JWT `admin_session` cookie authentication using `@/lib/auth`.
   - Read `data/docs.json`, filter out the document matching the `id`, and write it back.
   - Make a `fetch` request to `process.env.NEXT_PUBLIC_DELETE_WEBHOOK_URL` passing the `filename` or `id` so n8n can wipe the Vector Store. (Wrap in try/catch so it doesn't break if n8n is offline).
   - Return success.

2. **Update AdminPage.tsx**:
   - Add a "Delete" (Trash) icon button to each document row in the `AdminPage.tsx`.
   - Implement `handleDelete(id: string)`:
     - Set a loading state locally.
     - `fetch("/api/docs/" + id, { method: "DELETE" })`.
     - On success, filter the `docs` array in React state to remove the document layout immediately.

## Verification
- Run `npm run dev`. Navigate to `/admin`.
- Click the delete icon on a document.
- Verify the document disappears from the UI without a full reload.
- Verify `data/docs.json` no longer contains the document.
