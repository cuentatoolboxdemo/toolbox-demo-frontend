# Phase 4 Plan 2: Server Actions & Webhook Obfuscation

<objective>
Move n8n API calls to internal API routes to hide webhook URLs from the browser. Remove `localStorage` usage for System Prompt and centralize it. Filter chat history errors before server calls.
</objective>

<tasks>
1. **Create Chat API Route:**
   `type="auto"`
   Create `src/app/api/chat/route.ts`. It takes `{ question, tenant, history }` from the client, reads the global system prompt (from an env var or a local server-side config file), and makes a POST to the real n8n webhook.

2. **Create Ingest API Route:**
   `type="auto"`
   Create `src/app/api/ingest/route.ts`. It receives FormData with a file from the client and forwards it to the n8n ingest webhook. Secure this route by verifying the admin JWT cookie (from Plan 1).

3. **Refactor UploadZone:**
   `type="auto"`
   Update `UploadZone.tsx` to POST to `/api/ingest` instead of the public n8n URL.

4. **Refactor ChatInterface & History:**
   `type="auto"`
   Update `ChatInterface.tsx` to POST to `/api/chat`. Before sending the `history` array, filter out any assistant messages that have content starting with "Sorry, something went wrong".

5. **Refactor SystemPromptEditor:**
   `type="auto"`
   Remove `localStorage` logic from both Admin and ChatInterface. Build a simple Next.js Server Action or API route to read/write the system prompt to a file (e.g., `process.cwd() + '/data/systemPrompt.txt'`) as a poor-man's persistent database for the demo. Update `SystemPromptEditor.tsx` to use this new endpoint.
</tasks>

<verification>
1. The Network Tab in DevTools no longer shows requests to external n8n URLs.
2. The SystemPrompt persists across browsers/devices because it's stored on the server.
3. Chat history ignores error messages.
</verification>
