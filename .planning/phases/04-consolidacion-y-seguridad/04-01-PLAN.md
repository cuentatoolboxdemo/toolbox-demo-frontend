# Phase 4 Plan 1: Admin JWT Authentication

<objective>
Replace the vulnerable `sessionStorage("adminAuth")` mechanism with a simple HTTP-only cookie JWT authentication to secure the `/admin` portal.
</objective>

<tasks>
1. **Create Auth Utilities:**
   `type="auto"`
   Create `src/lib/auth.ts` with `jose` (or `jsonwebtoken`) to sign and verify a minimal JWT for the admin session. Use a hardcoded `JWT_SECRET` for now, or fall back to a random string.

2. **Create Auth API Routes:**
   `type="auto"`
   Create `src/app/api/auth/login/route.ts` and `src/app/api/auth/logout/route.ts`. The login route should verify "demo123", generate a JWT, and set an HttpOnly cookie. The logout route clears the cookie.

3. **Refactor AdminPage.tsx UI:**
   `type="auto"`
   Modify `AdminPage.tsx`. Remove `sessionStorage` logic. Replace it with a login form that Posts to `/api/auth/login`.

4. **Add Middleware Shield:**
   `type="auto"`
   Create `src/middleware.ts` to block access to `/admin` routes if the JWT cookie is missing or invalid. Redirect to a login state or return 401 if it's an API call.
</tasks>

<verification>
1. User cannot access `/admin` content without a valid cookie.
2. Logging in with "demo123" sets an HttpOnly cookie and reveals the admin UI.
3. Refreshing the page keeps the admin UI visible.
4. Logging out destroys the cookie and shows the login form again.
</verification>
