  <action>
    Completely rewrite `AnimatedAvatar.tsx` to replace the ring-based animation with a particle constellation system using HTML5 Canvas for performance.

    **Keep the same component interface:**
    ```tsx
    interface AnimatedAvatarProps {
      isShrunk: boolean;
      isThinking: boolean;
      theme: TenantTheme;
    }
    ```

    **Architecture:**
    - Use a `<canvas>` element for particle rendering (performant, GPU-accelerated)
    - Overlay the brand logo on top of the canvas using an absolutely-positioned `<img>` or `<div>`
    - Use `useRef` for canvas ref and animation frame ref
    - Use `useEffect` with `requestAnimationFrame` loop for smooth 60fps animation
    - Clean up animation frame on unmount

    **Particle system design:**
    - 30-50 particles (limit for mobile performance; reduce to ~25 if `isShrunk`)
    - Each particle has: x, y, radius (1-4px), angle, orbitRadius, speed, opacity, color
    - Particles orbit the center at varying radii (creating depth layers)
    - 3 orbit layers: inner (30-50px), middle (60-90px), outer (100-130px)
    - Particle colors randomly chosen from theme palette: [theme.primary, theme.avatarGlow, theme.avatarPulse, theme.ring]
    - Particles have slight random wobble on their orbit (sinusoidal perturbation) for organic feel

    **State reactions:**
    - **Idle** (isThinking=false): Gentle drift. Particles orbit slowly (speed 0.002-0.008 rad/frame). Subtle glow. Calm, meditative feel.
    - **Thinking** (isThinking=true): Particles accelerate (speed x3), orbit radii pulse outward then contract (breathing effect), particle opacity increases, add a subtle trail effect (lower canvas clear alpha). Outer particles move faster than inner ones.
    - **Transition between states**: Smoothly interpolate speed and radius over ~30 frames (not instant snap)

    **Visual effects:**
    - Faint connecting lines between nearby particles (distance < 60px), opacity based on distance (closer = more visible). Creates the "constellation" / neural network look.
    - Subtle radial gradient glow behind the center (using canvas radialGradient with theme.avatarGlow at 0.1-0.2 opacity)
    - Particles have soft edges (use arc with slight shadow or gradient fill)

    **Center logo:**
    - Position an `<img>` element absolutely centered over the canvas
    - Use theme.logoUrl as src
    - Size: 48px when not shrunk, 32px when shrunk
    - White circular background with subtle box-shadow using theme.avatarGlow
    - If no logoUrl, show a pulsing dot (fallback, same as current behavior)

    **Shrink behavior (must match current):**
    - The parent container in MessageList.tsx handles the CSS scale transform (scale-50 when messages > 0)
    - Inside AnimatedAvatar, when `isShrunk`: reduce particle count, reduce orbit radii, simplify effects (fewer connection lines)
    - The canvas size stays 256x256 (the CSS transform handles visual shrinking)

    **Performance requirements:**
    - Use `devicePixelRatio` for crisp rendering on retina displays
    - Limit particle count based on `isShrunk` state
    - Use `requestAnimationFrame` (not setInterval)
    - Clear canvas with slight alpha (0.15) for subtle motion trails during thinking, full clear during idle
    - Avoid creating new objects in the render loop (pre-allocate particle array)

    **Important:** Keep `"use client"` directive. Keep the `eslint-disable @next/next/no-img-element` comment. The component must remain a drop-in replacement -- MessageList.tsx must not need changes.
