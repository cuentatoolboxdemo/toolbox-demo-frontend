# Phase 8 Plan 1: Tenant Isolated Admin & System Prompt

## Objective
Migrate the global `/admin` page to a dynamic `/admin/[tenant]` route, sharing the same password but utilizing distinct `localStorage` keys for the system prompt.

## Steps
1. Rename/move `src/app/admin/page.tsx` functionality into `src/app/admin/[tenant]/page.tsx` mapping the `params.tenant` string.
2. In `src/app/admin/page.tsx`, create a simple router or redirect to `/` with an error indicating a tenant is required (e.g., `notFound()` or a selector).
3. The password gate checks against `demo123` globally. (Store session auth token using a generic `adminAuth` or `adminAuth_[tenant]`).
4. Update `src/components/system-prompt-editor.tsx` to receive a `tenantId` string prop. Change the `localStorage` key from `"systemPrompt"` to `"systemPrompt_${tenantId}"`.
5. Update `src/app/api/chat/route.ts` OR the webhook POST call in `src/components/chat-interface.tsx` to read `"systemPrompt_${tenantId}"`.

## Verification
- Password `"demo123"` successfully grants access to `/admin/calzedonia` and `/admin/ide_marketing`.
- Entering a system prompt for Calzedonia does NOT bleed over when reading the prompt for IDE Marketing.
