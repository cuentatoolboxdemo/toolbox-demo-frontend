# Phase 8 Plan 2: Tenant Isolated Document Lists & Uploads

## Objective
Isolate the document visualization and upload targets to ensure each tenant perceives their workspace as private.

## Steps
1. Modify `src/app/admin/[tenant]/page.tsx` and the `UploadZone` component to pass the `tenantId` context.
2. While the documents are largely hardcoded mock data for V1.1 frontend validation, adjust the mock active documents view parameter so it shows different strings or tags (e.g., "demo-[tenant]-doc.pdf") for verifying isolation.
3. Pass `tenant` to `/api/ingest` (or direct formData POST) dropzone upload handler payload alongside the FormData so the n8n webhook can be adapted in the future.

## Verification
- When uploading a PDF in `/admin/sabores`, the formData payload network event contains `tenant: 'sabores'`.
- The active documents view reflects tenant-specific contexts without crossing wires.
