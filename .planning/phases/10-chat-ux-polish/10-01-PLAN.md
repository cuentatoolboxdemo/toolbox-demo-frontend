# Phase 10 Plan 1: Animated AI Avatar Component

## Objectives
Introduce a dynamic, visually engaging "AI Presence" to the chat interface. When the conversation starts, the user should see a breathing, pulsating element (like a Jarvis core or a Siri wave) that signals readiness. As the chat history fills up with messages, this element should gracefully get out of the way, shrinking into the header or fading out to avoid obstructing text.

## Implementation Steps

1. **Create the `AnimatedAvatar` Component**:
   - Create a new file `src/components/chat/AnimatedAvatar.tsx`.
   - Build a CSS/Tailwind-based breathing animation. Use composite layered rings using `div` elements, `animate-pulse`, and `scale` transitions. Avoid adding heavy dependencies like Framer Motion for now. Keep it lightweight and stunning.

2. **Integrate into `ChatInterface`**:
   - In `src/components/chat/ChatInterface.tsx`, compute a derived state or logic based on `messages.length`.
   - If `messages.length === 0`: Show the `AnimatedAvatar` at the exact vertical and horizontal center of the chat area, large size (`w-32 h-32` or similar).
   - If `messages.length > 0`: Move/animate the `AnimatedAvatar` to a miniaturized version (e.g., `w-8 h-8`) placed in the header, or fade it out altogether if moving it is too visually disruptive without Framer Motion. 
   - A smooth middle-ground without complex dependencies is wrapping it in a container with standard Tailwind `transition-all duration-700 ease-in-out` classes, shrinking its height/opacity based on the message count. 

3. **Interaction Modifiers**:
   - When `isLoading` is true (waiting for the AI's response), boost the animation intensity (e.g., faster pulse, glowing effect) of the avatar, regardless of its position/scale.

4. **Refine Responsiveness**:
   - Ensure the scrollable message container cleanly overlaps or pushes the avatar away so readability is never compromised.

## Verification
- Load an empty chat: Verify the avatar is large and centrally rotating/pulsing.
- Send a prompt: Verify the avatar increases its animation intensity while fetching data.
- Receive response: Check that the avatar properly scaled down and shifted away from the center to make room for the text history.
